ContiguousListSlice

A ContiguousListSlice is a list-like data structure, implemented via a reversed ArraySlice. It has performance characteristics similar to an array, except that operations on the beginning generally have complexity of amortized O(1), whereas operations on the end are usually O(count).

Because an ArraySlice presents a view onto the storage of some larger array even after the original array’s lifetime ends, storing the slice may prolong the lifetime of elements that are no longer accessible, which can manifest as apparent memory and object leakage. To prevent this effect, use ContiguousListSlice only for transient computation.

Discussion of this specific implementation is available here.

    • Parameter maxSplits: The maximum number of ContiguousListSlices to return, minus 1. If maxSplit + 1 ContiguousListSlices are returned, the last one is a suffix of self containing the remaining elements. The default value is Int.max.
